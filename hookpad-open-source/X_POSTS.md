# HookPad X Posts

## Post 1 - Build Log

We are building HookPad for the X Layer Build X Hackathon.

Most token launches get decided in the first few minutes by snipers and aggressive early wallets.

HookPad moves the launch rules into the pool itself.

Small buys pass.
Large early buys get blocked or charged more.
Extra launch fees go to LP protection.

Built around a Fair Launch Hook for Uniswap V4-style pools on X Layer.

## Post 2 - Demo Proof

HookPad now has real X Layer testnet purchase receipts.

MockUSDC -> HPAD launch buy passed.
Large MockUSDC -> HPAD buy reverted.
Open-market MockUSDC -> HPAD buy passed.

That is the whole point of the Hook: launch protection should be visible on-chain, not just described in a dashboard.

Demo and contracts are ready for review.

## Post 3 - Product Angle

The interesting part of Uniswap V4 Hooks is not only custom fees.

It is that market rules can move closer to the pool.

For new assets, that means a launch does not have to be pure first-block chaos.

HookPad uses the Hook path for adaptive fees, wallet limits, and LP protection.

## Post 4 - Technical Note

HookPad has three demo paths on X Layer testnet:

1. Small early buy succeeds
2. Large early buy is blocked
3. Open-market buy succeeds after protection ends

The dashboard links directly to the testnet receipts so judges can verify the behavior.
